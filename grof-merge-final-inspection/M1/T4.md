# Milestone 1: Backend & Data Pipeline (T4)

**Team:** Khalil, Siwar
**Duration:** 4 Weeks
**Goal:** Build a production-ready Backend with Docker, Session Management, and Trace Aggregation logic.

---

## 1. Background & Context

### The Data Problem
Profiling generates a LOT of data.
- **CPU Samples:** 100Hz * 10 threads = 1000 events/sec.
- **GPU Kernels:** Can be 10,000+ per second.
We cannot just "dump" this into a DB. We need to structure it efficiently.

### Architecture
- **Database:** PostgreSQL (for metadata/sessions) + TimescaleDB (optional, for time-series) or just optimized tables.
- **API:** FastAPI (Python).
- **Deployment:** Docker Compose. The entire backend stack must come up with one command.

---

## 2. Detailed Steps

### Week 1: Docker & Infrastructure
1.  **Docker Compose:** Create a `docker-compose.yml` that spins up:
    - PostgreSQL (Database)
    - FastAPI (Backend)
    - PgAdmin (Database UI, optional but helpful)
2.  **Networking:** Ensure the containers can talk to each other.
3.  **Persistence:** Configure Docker Volumes so data isn't lost when containers restart.

### Week 2: Advanced Data Modeling
1.  **Schema:** Design the schema for:
    - `Session`: (id, name, start_time, end_time, git_commit_hash, tags).
    - `CpuSample`: (session_id, timestamp, thread_id, stack_hash).
    - `StackFrame`: (hash, function_name, file_path, line_no). *Use hashing to avoid storing the same string 1 million times.*
    - `GpuEvent`: (session_id, name, start_time, end_time, stream_id).
2.  **Migration Tool:** Set up `alembic` to manage database migrations. Do not create tables manually.

### Week 3: API Logic & Aggregation
1.  **Ingestion:** Implement high-performance ingestion endpoints.
    - `POST /api/v1/sessions/{id}/cpu-samples` (Accepts batch JSON).
2.  **Aggregation (The Hard Part):** The frontend doesn't want 1 million samples. It wants a "Flamegraph Tree".
    - Implement an endpoint `GET /api/v1/sessions/{id}/flamegraph`.
    - **Logic:** Query the samples -> Reconstruct the tree -> Sum the durations -> Return the JSON tree structure `{ name: "root", value: 100, children: [...] }`.
    - *Do this calculation on the Backend, not the Frontend.*

### Week 4: Session Management & Integration
1.  **Session Control:** Implement `POST /start` and `POST /stop` endpoints.
2.  **Run Management:** API to list all past profiling sessions with metadata (duration, total GPU time).
3.  **Testing:** Write integration tests using `pytest` that spin up the DB, insert fake data, and verify the Aggregation logic returns the correct tree.

---

## 3. Evaluation Metrics

To pass Milestone 1, your deliverable must meet these criteria:

1.  **One-Command Start:** `docker-compose up` must bring up the whole system.
2.  **Data Efficiency:** The database must use **Stack Frame Deduplication** (storing strings only once).
3.  **Server-Side Processing:** The Flamegraph JSON must be generated by the Python backend, not the React frontend.
4.  **Robustness:** The API must handle invalid JSON or missing fields gracefully (400 Bad Request, not 500 Error).
