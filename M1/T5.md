# Milestone 1: Visualization & UI (T5)

**Team:** Siwar, Olga
**Duration:** 4 Weeks
**Goal:** Build a dynamic React Dashboard with Flamegraphs, Timelines, and real API integration.

---

## 1. Background & Context

### The User Experience
A profiler is only as good as its visualization. Users don't want to read JSON. They want to see:
1.  **"Where is the bottleneck?"** (Flamegraph)
2.  **"Is the GPU idle?"** (Timeline / Gantt Chart)
3.  **"How did this run compare to the last one?"** (Dashboard)

### The Stack
- **React:** UI Framework.
- **D3.js:** Low-level visualization (Flamegraph).
- **Chart.js / Recharts:** High-level charts (Line charts for metrics).
- **Axios / React Query:** For fetching data from the Backend.

---

## 2. Detailed Steps

### Week 1: Project Structure & Dashboard
1.  **Setup:** Vite + React + TypeScript (strongly recommended for complex data).
2.  **Routing:** Set up `react-router`.
    - `/`: List of recent runs (Dashboard).
    - `/run/:id`: Details of a specific run.
3.  **Dashboard UI:** Create a table listing "Sessions" (mock data for now) with columns: ID, Name, Date, Duration.

### Week 2: The Flamegraph Component
1.  **Integration:** Integrate a flamegraph library (e.g., `d3-flame-graph`).
2.  **Interactivity:**
    - **Zoom:** Click to zoom in.
    - **Search:** Highlight frames matching a string (e.g., "conv2d").
    - **Tooltip:** Hover to see full function name and duration.
3.  **Color Coding:** Implement a color scheme (e.g., "warm" colors for Python, "cool" colors for C++/System).

### Week 3: The Timeline View (Gantt Chart)
*This is critical for GPU profiling.*
1.  **Concept:** A horizontal timeline showing GPU kernels.
    - Row 1: Stream 1
    - Row 2: Stream 2
2.  **Implementation:** Use a library like `vis-timeline` or build a simple SVG-based timeline with D3.
3.  **Data Mapping:** Map the GPU events (Start, End, Name) to rectangles on the timeline.

### Week 4: Real API Integration
1.  **Connect to T4:** Stop using mock data. Fetch the list of sessions from `http://localhost:8000/sessions`.
2.  **Fetch Trace:** When clicking a session, fetch the flamegraph JSON from `http://localhost:8000/sessions/{id}/flamegraph`.
3.  **Loading States:** Handle network latency (Spinners/Skeletons).
4.  **Docker:** Create a `Dockerfile` for the frontend (Nginx serving the build).

---

## 3. Evaluation Metrics

To pass Milestone 1, your deliverable must meet these criteria:

1.  **Real Data:** The UI must fetch data from the Backend API (T4), not a local JSON file.
2.  **Two Views:** You must have both a **Flamegraph** (CPU) and a **Timeline** (GPU).
3.  **Navigation:** You can navigate from the Dashboard list to a specific Run detail page.
4.  **Search:** You can search for a function name in the flamegraph.
