## 1. Background & Context

### Why Benchmarking First?
Before we build a profiler that claims to be "low overhead" (<5%), we need to know what the "normal" performance is. If we don't measure the baseline accurately, we can't prove our tool works.

### The Challenge of Variance
Computers are noisy. A model might take 100ms one run and 105ms the next. If our tool adds 5ms overhead, is that the tool or just noise?
You need to implement **statistical rigor**. We need Mean, Standard Deviation, and Confidence Intervals.

### Workloads
- **Macro-benchmarks:** Full models (ResNet, BERT).
- **Micro-benchmarks:** Specific operations (Matrix Multiplication, Convolution, LSTM cell). These are crucial for testing specific CUPTI events.

## 2. Detailed Steps

### Week 1: The Suite Structure
1.  **Repository Setup:** Create a clean structure in `benchmarks/`.
2.  **Micro-benchmarks:** Write scripts to isolate specific GPU behaviors:
    - `micro_gemm.py`: Pure Matrix Multiplication (Compute bound).
    - `micro_memcpy.py`: Heavy Host-to-Device transfers (Bandwidth bound).
    - `micro_launch.py`: Launching 10,000 tiny kernels (CPU overhead bound).
3.  **Macro-benchmarks:** Set up ResNet50 and BERT (HuggingFace) with fixed random seeds for reproducibility.

### Week 2: Automation & CI
1.  **Runner Script:** Create `run_suite.py`. It should:
    - Accept a list of models to run.
    - Accept a number of iterations (N).
    - Handle "Warmup" automatically.
    - Save results to a structured JSON/CSV file.
2.  **Environment Control:** The script should log the environment (GPU driver version, PyTorch version, OS) to ensure comparisons are valid.

### Week 3: Deep Profiling (The "Ground Truth")
1.  **Nsys Deep Dive:** Profile all benchmarks with `nsys`.
2.  **Metric Extraction:** Don't just look at the GUI. Use `nsys stats --report gputrace` to export CSVs.
3.  **Analysis:**
    - What is the average kernel duration for ResNet?
    - What is the "Gap" between kernels (CPU overhead)?
    - This data will be the "Gold Standard" we compare GROF against.

### Week 4: Statistical Analysis & Regression Framework
1.  **Variance Analysis:** Run the baseline 50 times. Plot the distribution of execution times.
2.  **Regression Test:** Write a script `check_performance.py` that takes two result files (Baseline vs New) and performs a T-test to see if the difference is statistically significant.
3.  **Documentation:** A detailed report on the "Performance Characteristics of PyTorch Models".

---

## 3. Evaluation Metrics

To pass Milestone 1, your deliverable must meet these criteria:

1.  **Automation:** `python run_suite.py` must run all tests and generate a report without human intervention.
2.  **Micro-benchmarks:** You must have at least 3 micro-benchmarks targeting different GPU bottlenecks.
3.  **Statistical Rigor:** Your reports must include Standard Deviation. "It takes 100ms" is not an answer. "100ms ± 2ms" is.
4.  **Data Export:** Results must be saved in a machine-readable format (JSON/CSV) for future plotting.
